# AnyPcc-Fast

This is an optimized inference implementation of AnyPcc. It keeps the same main API as AnyPcc, while replacing several bottlenecks in entropy coding, sparse hierarchy construction, and lossy/tune execution.

## Inference Optimizations


- Use the C++ CPU range coder in `kit/fast_torchac.py` instead of the Python `torchac` path.
- Use a torch scatter/index implementation for FOG stride-2 occupancy downsampling, while preserving z-order output and avoiding extra sorting.
- Use batched lossless-prefix encoding to reduce Python dispatch and sparse-convolution launch overhead.
- Use fp16 sparse convolution for non-tune inference, with outputs converted back to fp32.
- Implement `FCG` with `torch.nonzero` and `index_select`, avoiding large intermediate tensors from `repeat(1, 8).reshape(...)`.
- Reuse the child index generated by `FCG` in `TargetEmbedding`, avoiding recomputation from coordinates.
- In lossy mode, reuse the same optimized lossless prefix path. For the lossy tail, the encoder only writes `k`, avoiding unnecessary lossy-network forward passes.


Lossless end-to-end speedup:

| Implementation | BPP      | Enc avg | Dec avg | Total   |
| -------------- | -------- | ------- | ------- | ------- |
| `src`          | 0.498071 | 0.2997s | 0.2842s | 0.5839s |
| `src_fast`     | 0.498153 | 0.1260s | 0.1433s | 0.2693s |


- Encoding: `2.38x`
- Decoding: `1.98x`
- End-to-end encoding + decoding: `2.17x`

Lossy end-to-end speedup:


| Lossy level | `src` total | `src_fast` total | Total speedup |
| ----------- | ----------- | ---------------- | ------------- |
| 1           | 2.9142s     | 0.1482s          | 19.66x        |
| 2           | 2.8118s     | 0.1154s          | 24.36x        |
| 3           | 2.2860s     | 0.0986s          | 23.20x        |


## IAFT Optimizations


- Pack the four stage features from each depth into four large batches, so each optimizer step only runs four MLP heads.
- Remove per-step `requires_grad_()` and `clone()` calls on cached features. Gradients are only computed for prediction-head parameters.
- The tune-mode decoding time includes loading the sample-adaptive prediction-head parameters from the bitstream. Tune mode also keeps sparse convolution in fp32, while normal non-tune inference uses fp16 sparse convolution by default.


| Implementation      | Tune avg | Enc avg | Dec avg | Enc+Dec |
| ------------------- | -------- | ------- | ------- | ------- | 
| Before optimization | 2.4328s  | 2.6865s | 0.2325s | 2.9190s | 
| After optimization  | 0.9264s  | 1.1849s | 0.2324s | 1.4173s |


Tune-mode speedup:

- Tune stage: `2.63x`
- Total encoding time: `2.27x`
- End-to-end encoding + decoding: `2.06x`
